# Project Log - 2026-03-23

## Progress

- Updated "Agents only" filter default value behavior to match "Exclude deactivated users" pattern
- Default value changed to `false` (was previously `true` in initial implementation)
- Filter now only appears on filter bar when toggled to `true`; disappears when toggled back to `false` (default)

## Details

### Filter Bar Behavior Change

**Pattern to match:** "Exclude deactivated users"
- Default value: `false`
- When toggled to `true` → appears on filter bar
- When toggled back to `false` (default) → disappears from filter bar

**Applied to:** "Agents only" filter
- Previous design: default `true` (from Phase 3.1, 4.1 in implementation plan)
- New behavior: default `false`
- Consistent with standard filter bar pattern for optional filters

### Rationale

Having "Agents only" default to `false` makes the filter opt-in rather than opt-out:
- Users see agent + manager data by default
- Filter appears on bar only when explicitly excluding managers
- Cleaner filter bar when using default settings
- Aligns with "Exclude deactivated users" UX pattern

### Implementation Impact

This affects:
- Initial state values in `usePerformanceFilters.tsx`, `useLeaderboardsFilters.tsx`, `useAssistanceFilters.tsx`
- Default fallback values (`?? false` instead of `?? true`) in API call sites
- Filter bar visibility logic (filter chip only shown when `listAgentOnly === true`)
- LocalStorage defaults for new users
