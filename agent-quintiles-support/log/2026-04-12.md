# Project Log - 2026-04-12

## Progress

Fixed two AutoQA bugs in CriteriaLabeledOptions and TemplateBuilderAutoQA:

**Fix 1: N/A label in AutoQA dropdowns**
- Bug: N/A option showed "N/A (2)" instead of "N/A (no score)" when score is null
- Root cause: `label: ${option.label} (${score ?? option.value})` at TemplateBuilderAutoQA.tsx:242
- Fix: Check for `option.isNA && score == null` to show "no score" instead of falling back to option.value
- Commit: dd96747dc4

**Fix 2: Score auto-population when disabling AutoQA**
- Bug: When unchecking "Automated scoring" on new criteria, scores auto-populate as 0, 1, ... instead of staying empty
- Root cause: Legacy scores migration at CriteriaLabeledOptions.tsx:94-96 was running inappropriately
- Fix: Remove legacy scores array migration, keep only N/A option migration (which simulates "Allow N/A" checkbox for old templates)
- Commit: 99aaed9705

## Details

### Investigation Process

Created comprehensive analysis documents in `/director/.tmp`:
- `AUTOQA_SYNC_ANALYSIS.md` - Explained validation timing and sync mechanism

Key insights:
- React-hook-form's `required` rule checks existence, not truthiness (0 passes, null/undefined fail)
- useWatch + useMemo provides automatic sync from CriteriaLabeledOptions to AutoQA dropdowns
- Two types of migration: N/A option (keep) vs scores array (remove)
- Frontend shouldn't handle scores array migration (should be backend/one-time script)
