## Details

### Initial Hypothesis (Incorrect)

Initially suspected heartbeat detection failures (false negatives) due to network interruptions, based on:
- 92% of conversations on March 16 have `has_agent_assistance=false`
- Only 47 out of 568 active agents detected as logged in
- Documentation describes "false negatives" from network issues

### Critical Discovery: Direct Heartbeat Data

Analyzed `auth_mm_ohio.user_event_d` table in ClickHouse events DB:

**March 16, 2026**:
- Total agents with conversations: 603
- Agents with heartbeat events: **70** (11.6%)
- Agents with ZERO heartbeats: **533** (88.4%)
- Agents tagged with AA: 47 (7.8%)

**Heartbeat quality for the 70 agents**:
- Excellent coverage: 40-97 heartbeats per day
- Full work day spans: 11am-11pm (some 24-hour)
- Proper frequency: ~15-20 minute intervals

### Revised Root Cause

**LOW DESKTOP APP ADOPTION** - not heartbeat detection issues

- 88-90% of Med Mutual agents are NOT using the Desktop app at all
- For agents who ARE using it, heartbeat detection works properly
- The 67% tagging efficiency (70 heartbeats → 47 tagged) is a secondary issue

### Pattern Over Time

| Date | Heartbeat Coverage | AA Tagging Rate | Trend |
|------|-------------------|-----------------|-------|
| 3/10 | 10.3% | 2.9% | Baseline low adoption |
| 3/16 | 11.6% | 7.8% | Issue filed |
| 3/17 | 16.1% | 9.9% | Improving |
| 3/19 | 17.6% | 11.5% | Continued improvement |

Something changed around March 16-17 (deployment? training push? monitoring?), causing Desktop app adoption to increase.

### Why "Assistance Used" ≠ "Active Days"

Customer sees "assistance used" but no Active Days because:
- **Assistance used** = Conversation events (hints, KB) from CCaaS or post-call jobs
- **Active Days** = Desktop app heartbeats only

These are **independent data sources**. Agents can have conversations with assistance events without logging into Desktop app.

## Lessons Learned

**Always check the source data directly!** Initial hypothesis was based on symptoms (low AA tagging), but direct heartbeat event data revealed the true root cause (low adoption). The documentation's "false negatives" scenario is real, but it wasn't the primary issue here.

**Always verify example data points**: The agent mentioned by the customer (Iesha Williams) turned out to not be an active agent during the investigation period. Always verify specifics rather than assuming they represent the general case.
