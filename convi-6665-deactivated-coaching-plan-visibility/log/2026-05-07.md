# 2026-05-07

## Summary

Created the `convi-6665-deactivated-coaching-plan-visibility` project and completed the first-pass investigation across `director` and `go-servers`.

## What Moved

- Verified that the shared Director user/team/group picker already hides deactivated users by default but allows them to appear during search.
- Verified that Coaching Hub request builders do not preserve that behavior in backend requests:
  - recent activities request omits `includeInactiveUsers`
  - target progress request hard-codes `includeInactiveUsers: false`
- Verified that `go-servers` coaching overview/progress endpoints map `includeInactiveUsers=false` to `State=ACTIVE`, which filters selected deactivated users out.
- Verified that `ListCoachingPlans` does not currently set user `State` in its parsed user filter, so the active-only mismatch is not centered there.

## Current Read

- Most likely implementation path: FE change in Coaching Hub request builders.
- Possible BE follow-up only if the ticket requires "specific searched users only" even when mixed with team/group filters.
