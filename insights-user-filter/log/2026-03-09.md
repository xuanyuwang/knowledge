# Project Log - 2026-03-09

## Progress

- Created comprehensive comparison of all potential solutions for passing large user ID lists in ClickHouse queries

## Details

Documented 10 solutions in `solutions-for-large-user-id-lists.md`:

1. `ShouldQueryAllUsers` flag (already implemented)
2. `ext` external tables (chosen best general solution)
3. Increase `max_query_size` server setting
4. Temporary tables (INSERT then JOIN)
5. Query batching (split into chunks)
6. Reference table (persistent filter table)
7. User count threshold
8. Nil vs empty FinalUsers semantics
9. Per-query `max_query_size` setting
10. JOIN with inline VALUES / arrayJoin

Solutions 1 and 2 are the recommended path. Solution 1 handles "all users" cases, Solution 2 (`ext`) handles all remaining large-subset cases.
