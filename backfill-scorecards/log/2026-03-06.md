- **Mismatch: 183 (10.3%)** — ALL explained by N/A or exclude_from_qa edge cases

The mismatches are in fields that don't affect QA score computation:
- For exclude_from_qa: Python returns percentage=NULL (Go also returns nil from `ComputeCriterionPercentageScore`; historic had non-zero values likely from a different code version)
- For N/A: Python returns weight=0 (Go also returns nil; historic had non-zero weight, but weight is irrelevant when percentage is NULL)

### Script refactoring

Replaced `fetch_historic_scores()` with:
1. `fetch_director_scores()` — fetches raw scores from `director.scores`
2. `fetch_template_revisions()` — fetches and caches parsed template JSON
3. `compute_scores_for_scorecard()` — computes percentage_value, weight, float_weight, max_value, manually_scored using validated Python scoring logic

This removes the dependency on `historic.scorecard_scores` which was too slow on Oportun (no index on scorecard_id).
