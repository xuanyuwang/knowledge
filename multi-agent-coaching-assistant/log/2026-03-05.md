## Details

### Focus Criteria Fix

Previously, `_RECOMMENDATIONS` had `focus_criteria` hardcoded per recommendation, duplicating data already in `_CRITERION_ASSESSMENTS`. Now `get_mock_recommend_direct_response()` derives `focus_criteria` dynamically from each agent's `criterion_assessments` (filtered to `is_below_target`). This ensures recommendations always reference the same criterion data as the assessments.

### Full Resource Name Support

The frontend sends full resource names like `customers/cresta-sandbox-2/users/2026hackathon-1`, not bare keys like `2026hackathon-1`. Added `_mock_agent_key()` helper that extracts the user ID suffix for mock agent lookup, and propagates the full resource name into assessment and recommendation `agent_name` fields.
