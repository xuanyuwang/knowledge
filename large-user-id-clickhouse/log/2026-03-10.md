### Design Review Doc

**Notion:** https://www.notion.so/31f4a587b0618196a67fefc7cae0aba9
**Title:** Efficient Reference Data Filtering in ClickHouse Analytics Queries

Key framing decisions:
- Problem is not just "user ID list too large" — it's that our analytics query layer lacks a mechanism to incorporate reference data (users, scorecards, conversations) that lives outside ClickHouse
- Three structural constraints explain why: can't preload (data changes), must separate (ownership boundaries), can't post-process (aggregation happens in CH)
- ext tables are the solution because: easy to adopt, generalizable beyond users, good performance, minimal blast radius
- Future leverage: same pattern applies to scorecard IDs, conversation IDs, any reference data
