## Progress

## Details

### Key Finding: Different Timestamp Columns

The two APIs apply the time range filter to **different timestamp columns**:

| API | Column | Meaning |
|-----|--------|---------|
| `RetrieveQAScoreStats` (Performance) | `scorecard_time` in CH | Conversation start time |
| `RetrieveDirectorTaskStats` (QM Report) | `submitted_at` in PG | Scorecard submission time |

A scorecard submitted days after the conversation will appear in different time windows on each page.

### Additional Differences Found
- Different data sources (ClickHouse vs PostgreSQL)
- QM Report uses versioned template names (`@revision` suffix)
- QM Report filters through director tasks first (task_status IN [2,4])
- Time range boundary operators differ (`< end` vs `<= end`)

### Code Trace
- `scorecard_time` definition: `go-servers/apiserver/sql-schema/protos/dataplatform/coaching/scorecard.proto:45-48`

  - Process-based → scorecard create time
