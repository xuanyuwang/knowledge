### Key Insight

The general `handleRemoveOption` function already handles the N/A case through its remapping logic (lines 155-159):

```tsx
const notApplicable = autoQANotApplicableField.value as number | null;
if (notApplicable != null) {
  const newNotApplicable = remapIndex(notApplicable);
  autoQANotApplicableField.onChange(newNotApplicable === -1 ? null : newNotApplicable);
}
```

When the deleted option **is** the N/A option, `remapIndex(notApplicable)` returns `-1`, which sets `auto_qa.not_applicable` to `null`. This means we don't need a separate `removeNAOption` function - the general deletion logic already handles it correctly.
