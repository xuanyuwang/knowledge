## Details

- The Agent tab needs two distinct data jobs:
- aggregate scorecard counts for all agents at once
- detailed scorecards for a single agent, grouped by template, only when the drawer is opened
- `RetrieveQAScoreStats` remains the best fit for the aggregate table because it matches the current QA score semantics and filter surface of the Agent tab.
- `RetrieveQAConversations` remains the best fit for the on-demand drawer because it exposes QA scorecard detail rows that FE can group by template.
- `RetrieveScorecardStats` and `ListScorecards` can each solve parts of the problem, but they are less aligned with the current Agent-tab semantics and filter parity.
- For Manager tab consistency, the main table should stay on `RetrieveScorecardStats`.
- For Manager drawer consistency, use `ListScorecards` for a single manager with `creatorUserNames`, `startSubmitTime`, and `endSubmitTime`, then group by template on FE.
- The current Manager `Scorecards completed` metric means distinct submitted scorecards attributed to `creator_user_id`, not generic reviewed scorecards and not necessarily `submitter_user_id`.
- `RetrieveQAConversations` can filter submitted scorecards via `scorecard_statuses = MANUALLY_SUBMITTED`, which becomes `scorecard_submit_time <> 0`.
- `RetrieveQAConversations` does not apply `scorecard_reviewer_audience` in the traced ClickHouse path, does not filter by `creator_user_id` or `submitter_user_id`, and does not use `scorecard_submit_time` as the default time range basis.
