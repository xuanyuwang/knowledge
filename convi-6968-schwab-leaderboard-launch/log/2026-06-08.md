- Final backend semantics:
  - `QAAttribute.users/groups` remain agent filters on `agent_user_id`.
  - `QAAttribute.scorecard_reviewer_audience` is the scorecard submitter filter on `submitter_user_id`.
  - `QA_ATTRIBUTE_TYPE_AGENT` groups by `agent_user_id`.
  - `QA_ATTRIBUTE_TYPE_SCORECARD_SUBMITTER` groups by `submitter_user_id`.
  - `QA_ATTRIBUTE_TYPE_GROUP` remains agent-group aggregation; submitter-group aggregation is out of scope.
- Nil submitter audience means no submitter filter and preserves backward compatibility. Explicit empty submitter audience resolves all allowed submitters. Explicit resolved-empty submitter audience returns an empty response.
- The old submit-time parity gap is no longer a blocker because the latest Manager migration decision accepts QA API time-range semantics instead of preserving the old `RetrieveScorecardStats` submit-time behavior.
- Cleaned the completed proto and go-servers CONVI-6968 worktrees and local branches. Left the FE worktree in place because it still had local/ahead changes.
