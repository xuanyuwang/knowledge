## Details

### Key Finding: CH Query Aggregation

The golden SQL files show clearly that `GroupByAgent` (quintile path) does `GROUP BY agent_user_id` with no `criterion_id` — scores are pre-aggregated across all criteria at query time. This means negation MUST happen in the CH query, not in Go post-processing.

### Approach: Option 1 — Ranking Column

- Keep `weighted_percentage_sum` unchanged for display (FE shows this score)
- Add `ranking_weighted_percentage_sum` that negates lower-is-better criteria
- Use ranking score only for quintile assignment sort
- Carry ranking score via side map (`map[agentUserID]float64`), not proto field

### WithMoment / WithMetadataView Complication

These query variants have two levels:
1. Inner CTE `scorecard_score_per_conversation`: per (conversation, scorecard)
2. Outer SELECT: `SUM(weighted_percentage_sum)` across conversations

Both levels need the ranking column propagated.
