# 2026-06-04

## ClickHouse `scorecard_time` trace

- Traced ClickHouse `scorecard_time` back to its Postgres source fields.
- For conversation scorecards, `scorecard_time` is derived from Postgres `chats.started_at`.
- For process scorecards, `scorecard_time` is derived from Postgres `scorecards.process_interaction_at` when valid and non-zero, otherwise the default ClickHouse sentinel time is used.
- Confirmed `scorecard_time` is distinct from submission time. Submission time is stored separately as ClickHouse `scorecard_submit_time`, sourced from Postgres `scorecards.submitted_at`.
- Leaderboard implication: QA APIs that use the default score/scorecard time path filter by `scorecard_time`, which means conversation start time for conversation scorecards. Current Manager `Scorecards completed` uses `RetrieveScorecardStats`, which rewrites the time filter to `scorecard_submit_time`.
