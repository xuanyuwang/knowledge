## Details

### i18n consolidation

**Before:** Three duplicate keys across two namespaces:
- `director-app-insights` → `leaderboards.filters.agents-only` = "Agents only"
- `director-app-insights` → `performance.filters.agents-only` = "Agents only"
- `director-app-insights` → `performance.filters.agents-only-yes` = "Yes"
- `director-app-insights` → `performance.filters.agents-only-no` = "No"

**After:** Single shared key following `exclude-deactivated-users` pattern:
- `director-app` → `common.agents-only-filter` = "Agents only"
- Yes/No labels reuse existing `common.boolean-filter.yes` / `common.boolean-filter.no`

**Files changed:**
- `director-app.json`: added `common.agents-only-filter`
- `director-app-insights.json`: removed 4 duplicate keys
- `useLeaderboardsFilters.tsx`: `tInsights('agents-only')` / `t('agents-only')` → `tCommon('agents-only-filter')`, removed unused `t` hook + dep
- `usePerformanceFilters.tsx`: `t('agents-only')` → `tCommon('agents-only-filter')`, `t('agents-only-yes/no')` → `tCommon('boolean-filter.yes/no')`, removed unused `t` hook + dep

### Lint fix

Removing the `t` translation hook left stale `t` entries in `useMemo` dependency arrays. ESLint `react-hooks/exhaustive-deps` flagged these as unnecessary dependencies. Removed from both files.
