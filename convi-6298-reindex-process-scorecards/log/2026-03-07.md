## Details

### reindexConfig struct
Extracted all input resolution into `buildReindexConfig()` which returns a single `reindexConfig` struct. Sources: proto config > env var flags > defaults. All downstream code reads from this struct only ‚Äî no more scattered `config.X` and `*FlagName` references.

### Batch-fetch director scores
Changed from 1 PG query per scorecard to 1 per batch: `SELECT * FROM director.scores WHERE customer=? AND profile=? AND scorecard_id IN (...)`. Groups results in memory by scorecard_id.

### CH deletion strategy
Moved from per-scorecard DELETE (2N mutations per batch) to single upfront DELETE for the entire job:
```sql
DELETE FROM score_d ON CLUSTER conversation
  WHERE scorecard_template_id IN (?) AND scorecard_time >= ? AND scorecard_time < ?
```
- Uses distributed tables (`score_d`, `scorecard_d`) with `ON CLUSTER conversation`
- 2 total mutations regardless of data volume
- Per-batch approach would create ~1720 mutations for 43K scorecards, risking CH mutation queue limits
