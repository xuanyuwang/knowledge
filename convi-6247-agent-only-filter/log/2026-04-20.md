## Details

### Missed components (follow-up to PR #17931)

PR #17931 (20-file fix) missed two leaderboard components that also call `useInsightsRequestParams` without `filterOptions`:

1. **`SummaryUsedLeaderboardByType.tsx`** — Summary tab leaderboard, 2 calls
2. **`GenAIAnswersLeaderboardByType.tsx`** — GenAI Answers tab leaderboard, 2 calls

Both already receive `filterOptions` via parent `{...props}` spread from `AssistanceInsightsCommonPageProps`, but the child props interface didn't declare it, so it was silently dropped.

Fix: Added `filterOptions?: FilterByAttributesOptions` to both component props, destructured from props, and passed as 5th arg to `useInsightsRequestParams`.

### Bottom-up API call trace

Did a full bottom-up trace from actual API/RPC calls back to components to verify coverage. This is more reliable than the top-down approach (tracing `filterOptions` from parent → child) that missed the two components above.

**Why top-down scanning misses things:**
- Top-down traces `filterOptions` from page container downward. If a child component's TypeScript props interface doesn't declare `filterOptions`, the prop is silently dropped — no type error, no visible gap.
- Components using different prop names (e.g. `commonFilterValues` instead of `filterValues`) or wrapping with `useUsecasesFilter()` look different from the pattern being searched.
- Components that bypass `useInsightsRequestParams` entirely (calling API hooks directly) are invisible to a hook-level scan.

**Bottom-up approach (recommended for verification):**
1. Find ALL API calls made by the page (grep for `CrestaAPI.insights.*`, `useQuery`, service calls in the hooks directory)
2. For each API call, check if `filterToAgentsOnly` is in the request payload
3. Trace backward: hook → which components call it → does the component pass `filterOptions`

**Results — 14 API calls traced:**

|---|------|------------|--------------------|
| 1 | useAssistanceStats | retrieveAssistanceStats | YES |
| 2 | useAgentStats | retrieveAgentStats | YES |

All 13 stats APIs pass `filterToAgentsOnly` via `InsightsRequestParams`. The only exception is `retrieveConversationMessages` (hint conversation examples drawer), which uses `CustomerInsightsRequestParams` and doesn't support the field.

**Two layers where gaps can occur:**
1. **Hook → API**: All hooks correctly pass `filterToAgentsOnly`. No gaps.
2. **Component → Hook**: This is where gaps were — components not passing `filterOptions` to hooks, so `filterToAgentsOnly` defaults to `false`. Fixed by PR #17931 (20 files) + PR #18132 (2 files).
