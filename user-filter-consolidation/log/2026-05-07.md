# Project Log - 2026-05-07

## Progress

- Captured a new future consideration from CONVI-6665 around deactivated-user handling.
- Recorded the short-term decision to support only explicitly searched users for coaching.

## Details

### New design pressure on shared user-filter semantics

Current shared user-filter state controls are coarse-grained and apply to the full parse result. That is sufficient for current callers, but it does not cleanly support a mixed policy such as:

- explicitly selected users can bypass active-state filtering
- users returned through team/group expansion remain active-only

### Current decision

For the coaching ticket, the narrow product decision is:

- support only the specifically searched user case for now
- defer any generalized source-aware state model

### Why this belongs in user-filter-consolidation

If this requirement appears in multiple surfaces, it should likely be solved once in the shared filter API rather than as ad hoc caller logic. The project README and implementation considerations were updated accordingly.
