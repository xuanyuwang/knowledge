### Behavior Gap Found

Testing on staging revealed that `retrieveQAScoreStats` with no user/group filter still returned unknown users (userId "0"). Investigation showed the ext table was never created in this case.

**Root cause:** `ShouldQueryAllUsers` flag in `ApplyUserFilterFromResult`

The original design had two independent concerns:
1. **`ShouldQueryAllUsers`** — When true (root access + empty filters), skip the user WHERE clause entirely to avoid query size limits with large user counts
2. **ext table flag** — When true, use ext tables instead of IN clauses for user filtering

The gap: when `ShouldQueryAllUsers=true`, `ApplyUserFilterFromResult` set users to `[]`, so the ext table was never built — even though the ext table eliminates the query size concern that `ShouldQueryAllUsers` was designed to work around.

**Before (PR #26178):**
```
ShouldQueryAllUsers=true  → users=[]  → no WHERE clause  → returns ALL rows in CH (including unknown users)
ShouldQueryAllUsers=false → users=FinalUsers → WHERE IN(...) or ext table
```

**After (this fix):**
```
ShouldQueryAllUsers=true  + extTable=true  → users=FinalUsers → ext table filters precisely
ShouldQueryAllUsers=true  + extTable=false → users=[] → no WHERE clause (legacy behavior)
ShouldQueryAllUsers=false → users=FinalUsers → WHERE IN(...) or ext table
```

### Code Changes

**`common_user_filter.go`** — `ApplyUserFilterFromResult` now accepts `useExtTable bool` parameter:
```go
func ApplyUserFilterFromResult(result *ParseUserFilterResult, users *[]*userpb.User, groups *[]*userpb.Group, useExtTable bool) bool {
    if result.ShouldQueryAllUsers && !useExtTable {
        *users = []*userpb.User{}
    } else {
        *users = result.FinalUsers
    }
    // ...
}
```

### Impact

With this fix, when the ext table flag is on:
- **All queries filter by the resolved user set** from `ParseUserFilterForAnalytics`, regardless of `ShouldQueryAllUsers`
- No more "unknown user" results from unfiltered CH queries
- The `ShouldQueryAllUsers` optimization (skip WHERE clause) is effectively superseded by ext tables
