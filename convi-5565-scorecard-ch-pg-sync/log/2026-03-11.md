### Separate Finding: ~67% of conversations never ingested into CH

Consistently across all 39 days, ~67% of PG submitted scorecards don't exist in CH at all (by scorecard_id lookup). These are NOT CONVI-5565 — the conversations were never ingested into ClickHouse.

#### Investigation

**Not a scorecard-type issue:**
- scorecard_type 4/5 (appeal): 100% missing — expected, filtered by CONVI-6227
- scorecard_type NULL (regular): ~65-68% missing across all templates equally

**All-or-nothing per conversation (confirmed at scale on March 10):**

| Category | Conversations | % |
|----------|--------------|---|
| All scorecards in CH | 137 | 34.8% |
| All scorecards missing | 257 | 65.2% |
| Mixed (some in, some not) | 0 | 0.0% |

Every template for a conversation is either all present or all absent. This rules out template-level or scorecard-level filtering.

**Conversations themselves don't exist in CH:**
- Missing conversations have 0 rows in CH `conversation` table
- Present conversations have 1 row in CH `conversation` table
- This is a conversation-level ingestion gap, not scorecard-specific

**Not a "reindex hasn't run yet" issue:**
| Conversation date | In CH | Missing | Miss% |
|-------------------|-------|---------|-------|
| 2026-03-02 | 0 | 1 | 100% |
| 2026-03-05 | 3 | 2 | 40% |
| 2026-03-06 | 3 | 1 | 25% |
| 2026-03-07 | 5 | 11 | 69% |
| 2026-03-08 | 13 | 26 | 67% |

| 2026-03-10 | 53 | 134 | 72% |

Conversations from March 5-8 (5+ days ago) are still missing at ~40-67%. Daily reindex should have covered them.

**CH write source analysis (for scorecards that DO exist in CH):**
- Feb 1–22: 0% from API async work — 100% from reindex
- Feb 23+: API async starts appearing (~5-10% of writes), rest from reindex
- API async work is silently failing for most scorecards

**Conclusion:** ~65% of Spirit conversations are never ingested into ClickHouse by either the reindex pipeline or API async work. This is a pre-existing ingestion pipeline gap, not related to CONVI-5565. Needs separate investigation.

### Timestamp Padding

All "submit time mismatches" across all days were microsecond zero-padding differences (e.g., `15:31:50.78114` in PG vs `15:31:50.781140` in CH). These are not real mismatches — just different precision formatting between PostgreSQL and ClickHouse DateTime64(6).
