# Project Log - 2026-03-09

## Progress

- Fixed 6 failing test suites after removing `enableParseUserFilterForAnalytics` feature flag

## Details

### Root Cause
After removing the feature flag, all code paths now run `ParseUserFilterForAnalytics`, which:
1. Calls `listAllUsers` to get ground truth users (IDs: `1`, `2`, `3`, `4`)
2. Calls `applyResourceACL` for access control
3. Intersects request user filter with ground truth users using full resource name as key

Two categories of test failures:

**Category 1: Wrong customer ID in user resource names**
Several test files used `customers/test_customer/users/1` but the test suite's `CustomerID` constant is `"customer_id"`, so ground truth users have keys like `customers/customer_id/users/1`. The intersection found no matches, returning empty results.

Fixed files:
- `retrieve_summarization_stats_test.go` - `Test_BasicFilterAndGroupByAgent`
- `retrieve_smart_compose_stats_test.go` - `Test_BasicFilterAndGroupByAgent`
- `retrieve_suggestion_stats_test.go` - `Test_BasicFilterAndGroupByAgent`
- `retrieve_hint_stats_test.go` - `GroupByAgentAndGroup`, `GroupByGroup`

**Category 2: Missing user metadata enrichment in expected response**
`retrieve_conversation_stats_test.go` - The new code path enriches users with `FullName` and `Username` from ground truth. The shared `testConvStatsHelper` expected response for agentUserA was missing these fields. But enrichment only happens when the user is in the ground truth set (i.e., when no user filter is specified, all users are enriched; when a specific user filter is applied, only those users are enriched).

Fix: Made the expected `agentUserAProto` conditionally include `FullName`/`Username` based on whether the request has a user filter.

**Category 3: Duplicate `configClient.Get()` mock expectations**
`retrieve_assistance_stats_test.go` had 5 test methods each setting up `mockConfigClient.EXPECT().Get(gomock.Any()).Times(1)`. But the base suite's `SetupSuite` (in `base_test.go`) already registers `Get().AnyTimes()`. The `AnyTimes()` expectation absorbs all actual calls, so the per-test `Times(1)` expectations were never consumed, causing gomock's `Finish()` to report "missing call(s)".

Fix: Removed all 5 redundant `Get().Times(1)` expectations from `retrieve_assistance_stats_test.go`. The `GetProfileConfig().Times(1)` expectations were kept since those are not covered by the base suite.
