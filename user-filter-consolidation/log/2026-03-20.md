## Key Decisions

### 1. Reuse proto types instead of custom types
**Decision**: Use `internaluserpb.LiteUser` and `internaluserpb.LiteGroup` from cresta-proto directly in `ParseResult`.

**Rationale**: No need to duplicate — proto types already have all fields we need (UserId, Username, FullName, Memberships / GroupId, DisplayName, GroupType). The resource name is already the map key, so no need to add it as a field in the value.

### 2. Interface name: `Parser` (not `UnifiedParser`)
**Decision**: Name the interface `Parser` with method `Parse(ctx, opts) (*ParseResult, error)`.

**Rationale**: 
- No conflict with existing `UserFilterParser` — different method signature
- Follows repo convention: `shared/gh.Client`, `shared/linear.Client`, `shared/sesv2.Client`
- Package qualifier makes it clear: `userfilter.Parser`
- Method name `Parse` is cleaner than `ParseV2` (different interfaces = no conflict)

### 3. Functional options pattern
**Decision**: Add `options.go` with `NewParseOptions()` and `Option` functions.

**Rationale**:
- **Coaching callers need it**: All 3 coaching callers set identical defaults (Roles=[AGENT], GroupRoles=[AGENT], UserTypes=[PROD,GUEST], State=ACTIVE)
- **Extensible**: Can add new defaults without breaking existing code
- **Consistent**: Same pattern across analytics and coaching callers

## Types Design

### ParseOptions
Organized by role:
- **Scoping**: CustomerID, ProfileID
- **Selection**: SelectedUsers, SelectedGroups (resource names)
- **Population filters**: ListAgentOnly, ExcludeDeactivatedUsers, IncludePeerUserStats
- **Membership resolution**: DirectMembershipsOnly
- **Legacy coaching filters**: Roles, UserTypes, GroupRoles, State (for ListUsers compatibility)

### ParseResult
Maps keyed by resource name, values are proto types:
- `FinalUsers map[string]*internaluserpb.LiteUser`
- `FinalGroups map[string]*internaluserpb.LiteGroup`
- `UserToDirectGroups map[string][]*internaluserpb.LiteGroup`
- `UserToAllGroups map[string][]*internaluserpb.LiteGroup`
- `AllGroups map[string]*internaluserpb.LiteGroup`
- `ShouldQueryAllUsers bool`

Note: `GroupToDirectMembers` / `GroupToAllMembers` omitted — derivable from user-to-group maps; add if a caller needs them.

### Parser interface
```go
type Parser interface {
    Parse(ctx context.Context, opts ParseOptions) (*ParseResult, error)
}
```
