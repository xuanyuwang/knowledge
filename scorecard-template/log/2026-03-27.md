## Details

### Scoring Approach Revision

**Problem**: `ComputeScores()` mutates score objects that are then persisted to DB via `UpdateScorecardAndScoresInDB()`. The original design (Spot 1) would flip `NotApplicable` to `false` and inject `NumericValue` — overwriting the grader's N/A selection in the database.

**Fix**: Handle NAScore in `ComputeCriterionPercentageScore()` instead. This function computes derived percentage values without modifying the stored score rows. DB preserves `not_applicable = true`, `numeric_value = NULL`. The configured NAScore is returned directly as a percentage.

### FE Requirements Added

1. **Current state documented**: N/A toggle only available for `labeled-radios` and `dropdown-numeric-values`, no config UI for N/A value/score
2. **New template builder behavior**: N/A row at bottom of options table with disabled "N/A" label, editable score input ("no score" placeholder), controlled by "Allow N/A" toggle
3. **Opera Integration sync**: NAScore reflected in auto-QA config section
4. **FE template parsing**: Read `settings.naScore` from API, serialize back on save
5. **Grader UI**: No changes (still submits `not_applicable: true`)
