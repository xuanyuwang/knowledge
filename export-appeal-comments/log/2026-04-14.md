  2. How scorecard data is queried for export
  3. How to query appeal approval reasons for scorecards
- Documented findings and drafted implementation plan in `research.md`

## Details

### Key Findings

- Comment column is populated from `criterionScores[0].Comment.String` in `convertScorecardsToCSVBytes()` (line 632 of `action_export_scorecards.go`)
- The proto `Score.comment` field is overloaded: on regular scorecards it's the grader comment, on appeal resolve scorecards it's the approval reason
- `NewGroupedAppealWorkflowsFromScorecardIDs()` in `shared/scoring/appeal_workflow.go` can fetch appeal workflows for a batch of scorecard IDs using a recursive CTE
- `FindExistingScores()` in `shared/scoring/scorecard_scores_dao.go` can fetch scores (including comments) for the resolve scorecards

### Implementation Approach

Add appeal approval reason lookup in `extractLinkedScorecardsDataForExport()`, then override the comment in `convertScorecardsToCSVBytes()` when an approval reason exists.
