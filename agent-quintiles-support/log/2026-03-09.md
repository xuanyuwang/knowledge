# Project Log - 2026-03-09

## Progress

- Investigated how outcomes (AHT, Conversion, CSAT) participate in overall agent quintile calculation for the leaderboard-by-criteria table
- Queried actual ClickHouse data for template `019b2dde` to verify `percentage_value` storage
- Documented findings in `outcome-quintile-investigation.md`

## Details

### Key findings

1. **Outcomes and criteria share the same scoring pipeline** — both stored in `score_d` / `scorecard_score_d` with `percentage_value` and `float_weight`
2. **`percentage_value` is NOT always normalized** — AHT stores raw seconds (59-2998), while dropdown items like CSAT use the template's `scores` mapping to produce 0/1
3. **`excludeFromQAScores` flag is the gate** — BE's `getScoreableCriteria()` filters out items with this flag before the ClickHouse aggregation query
4. **For template `019b2dde`**: AHT and Conversion excluded (`excludeFromQAScores: true`), CSAT included (mapped to 0/1 via scores config), 5 servicing criteria included (binary Yes/No)
5. **No validation safety net** — if `excludeFromQAScores` is misconfigured on a non-normalized item, raw values would corrupt the weighted average
6. **Explicit criterion ID bypasses filtering** — `getScoreableCriteria()` only runs when `criterionIdentifiers` is empty; explicit requests for excluded outcomes are not blocked (not a practical issue since FE never does this)
