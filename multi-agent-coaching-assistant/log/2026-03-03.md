### Context Switcher UI
- Collapsible bar at top of page showing current cluster/customer/profile
- 4 input fields (cluster, customer, profile, token) + Apply button
- `POST /api/set-context` leverages existing `context.py` after_request hook to set HTTP-only cookies
- Selecting a preset with context mismatch auto-fills fields and opens the panel
- Token field masked by default with Show/Hide toggle

### CORS & Header-Based Context
- CORS allowed origins: `*.cresta.ai`, `*.cresta.com`, `localhost` (regex-based allowlist)
- OPTIONS preflight handler for all `/api/*` routes
- `context.py` extended to read context from headers (priority: query params > headers > cookies):
  - `Authorization: Bearer <token>` → access_token
  - `X-Cluster-Id` → cluster_id, `X-Customer-Id` → customer_id, `X-Profile-Id` → profile_id
- Enables Director frontend to call endpoints directly via `fetch()` without cookie/same-origin constraints
- README updated with Director integration example (TypeScript)
