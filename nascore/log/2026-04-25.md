## Key Findings

### Value semantics shift across stages
`option.value` means different things at different stages — the load-time decoupling transform replaces it with the score value, while the save-time transform re-indexes back to 0,1,2... This round-trip works but is confusing.

### Latent bug: not_applicable not remapped on legacy load
In `transformApiCriterionTemplateSettingsToForm()`, the legacy migration path remaps `detected` and `not_detected` via `valueToIndexMap` but skips `not_applicable`. If a legacy template has `not_applicable` set, it won't be correctly remapped.

### Dual migration paths are redundant
Both `transformApiCriterionTemplateSettingsToForm()` (load-time) and `CriteriaLabeledOptions.useOnMount()` (mount-time) do legacy migration. The mount-time one is a safety net but makes the flow harder to trace.

### Renormalization logic is copy-pasted across 5+ handlers
Every mutation that touches options (add, remove, NA score change, NA score blur, allow NA toggle) independently does the same renormalize-and-remap dance.

## Refactoring Direction (merged with Codex design)

Cross-validated `options-scores-lifecycle.md` against `codex-refactor-design.md` (independently produced by Codex). Key reconciliations:

- **Adopted Codex's two-level module split**: shared read helpers in `director-api` + builder-side mutation module. Fixes wrong dependency direction (runtime scoring importing from template-builder).
- **Adopted Codex's canonical invariant**: "option.value is never the actual score in canonical form state" — `option.value = index`, `scores[i].score = actual score`.
- **Adopted Codex's DEFAULT_CRITERION fix**: add `scores` array so new criteria are canonical from the start, eliminating mount-time migration.
- **Kept save fallback risk** (not in Codex): `transformDropdownNumericInputOptionsToApi` uses `scores[i]?.score ?? option.value` — if canonical form has `option.value = index`, this fallback returns index instead of score. Must fix during refactor.
- **Kept not_applicable remap bug** (not in Codex): legacy load path remaps detected/not_detected but skips not_applicable. Fix before or during refactor.
- **Kept full grading path and value semantics table** — critical implementation reference not present in Codex doc.

Both documents agree on: core problem diagnosis, N/A lifecycle semantics, normalization at boundaries, pure-function approach, occurrence-based exclusion, no backend changes.

## Review feedback incorporated

Three issues fixed in `options-scores-lifecycle.md`:

1. **[P1] Section 1 invariant table was misleading** — claimed `options[i].value === scores[i].value` as current invariant, but the decoupled load path breaks this in form state. 3. **[P2] Missing mutation entrypoints** — section 3 claimed all mutations happen in CriteriaLabeledOptions or NumericBinsAndValuesConfigurator. Added: TemplateBuilderCriterionConfiguration.resetSettings() (resets options/scores/auto_qa on score type switch), TemplateBuilderScoreType.handleTypeFieldChange() (wipes settings on type change, resets range), TemplateBuilderFormConfigurationStep.handleAddCriterion() (initializes options/scores/auto_qa on creation/cloning). Added sections 3i and 3j to document these.
