## Details

### Missing `filter_to_agents_only` on prod for QA score stats

**Symptom:** `RetrieveConversationStats` sends `filter_to_agents_only: true` on prod, but `RetrieveQAScoreStats` omits it entirely.

**Root cause:** In PR #16777 (deployed), `modifiedFiltersState` sets `listAgentOnly: undefined` when `enableAgentOnlyFilter` flag is off:

```ts
if (!enableAgentOnlyFilter) {
  return { ...state, listAgentOnly: undefined };
}
```

Two different code paths handle this differently:
- `useInsightsRequestParams` → `filterOptions?.filterToAgentsOnly ?? true` → catches `undefined`, sends `true`
- `useQAScoreStatsRequestParams` → `filtersState.listAgentOnly` → passes `undefined` through, field omitted from request

**Fix:** PR #17356 (merged Mar 16, deployed to staging) changes the default from `undefined` to `true`:

```ts
if (!enableAgentOnlyFilter) {
  return { ...state, listAgentOnly: true };
}
```

**Impact:** No actual metric impact — BE still hardcodes `true` for QA score stats (BE PR #26301 not yet deployed). The field is just missing from the request payload.

**Status:** Fix confirmed working on staging. Will reach prod in next release cycle.
