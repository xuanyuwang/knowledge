# Project Log - 2026-03-17

## Progress

- Releaser had not promoted `00-head` to release stages overnight — flag was off everywhere after per-cluster patches were removed
- Created flux-deployments PR #264149 to manually add the flag to all 3 release stages (01-staging, 02-prod-early, 03-prod-main)

## Details

### Releaser Gap

PR #264076 (merged Mar 16) added the flag to `00-head` and removed per-cluster patches. However, the releaser only promotes on new image builds, not env var changes alone. This meant the flag was effectively **off on all clusters** after the per-cluster patches were removed — a regression from the previously working staging + voice-prod state.

### Fix: Direct Stage Updates

Rather than waiting for the releaser, added `ENABLE_EXT_TABLE_FOR_USER_FILTER: "true"` directly to:
- `apps/insights-server/releases/01-staging/helmrelease-insights-server.yaml`
- `apps/insights-server/releases/02-prod-early/helmrelease-insights-server.yaml`
- `apps/insights-server/releases/03-prod-main/helmrelease-insights-server.yaml`

**PR:** [flux-deployments#264149](https://github.com/cresta/flux-deployments/pull/264149)

### Lesson Learned

When moving a flag from per-cluster patches to app-level `00-head`, also update the stage files directly. Don't assume the releaser will propagate env-only changes promptly — it's tied to image builds.
