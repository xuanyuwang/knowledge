### Explaining the gaps

**PG 3,524 scorecards → CH 196 scorecards**: The activity only writes scorecard rows for scorecards that have at least one computed historic score (activity.go lines 220-224). Of 3,524 total:
- 1,766 have no `director.scores` rows at all (empty/unscored) → skipped
- 1,758 have director.scores, but only 196 produce valid historic scores after `GenerateHistoricScorecardScores` filters → the rest have scores for criteria that don't match the template structure (chapters, deleted criteria, or template parse failures on a test profile)

This is a test profile (walter-dev) with lots of experimental templates where criteria are frequently changed/deleted, so a high mismatch rate is expected.

**PG 2,412 scores → CH 721 scores**: `GenerateHistoricScorecardScores` matches each `director.scores` row against the parsed template structure. Only scores whose `criterion_identifier` is a leaf criterion in the current template revision produce a historic score row. Scores for:
- Non-leaf criteria (chapters/sections) → filtered out
- Criteria deleted from the template after scoring → no match, filtered out
- Template revisions that fail to parse → entire scorecard skipped

On a test profile with frequently-edited templates, many criteria in `director.scores` no longer match the template structure, causing a high filter rate.

**Key**: This behavior matches the existing `CreateScorecard` real-time path (Path 1 in investigation-sync-paths.md) which also uses `GenerateHistoricScorecardScores`. The reindex produces the same output as real-time creation would — this is correct behavior, not a bug.

## Bugs Found

### Bug 1: `deleteFromCH` uses distributed tables (BLOCKING)

**File**: `temporal/ingestion/reindexscorecards/activity.go:465`

```go
// BUG: attempts DELETE on distributed tables
for _, table := range []string{"score_d", "scorecard_d"} {
```

**Fix**: Use local tables with ON CLUSTER:
```go
for _, table := range []string{"score", "scorecard"} {
```

ClickHouse does not support DELETE on distributed tables. Must delete from local (ReplicatedReplacingMergeTree) tables with `ON CLUSTER 'conversations'`.

### Bug 2: Cron hardcodes `CleanUpBeforeWrite: true` (BLOCKING)

**File**: `cron/task-runner/tasks/batch-reindex-conversations/task.go:118`

```go
payload := &jobpb.ReindexScorecardsPayload{
    CleanUpBeforeWrite: true,  // hardcoded, no env var override
}
```

**Impact**: Every cron-triggered reindex hits Bug 1 and fails. Even after Bug 1 is fixed, cleanup-on-every-run is wasteful (CH `ReplacingMergeTree` already deduplicates). Should default to `false` or be configurable via env var like other settings.

### Script bug: Querying local tables instead of distributed

The `test-walter-dev.sh` verify command was querying `scorecard` (local) instead of `scorecard_d` (distributed). Process scorecards are sharded by `process_interaction_at % shard_count`, so data lands on different shards. Fixed to use `scorecard_d`/`score_d` for all CH queries.
